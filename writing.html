Field note

Traceability is a product requirement for production RAG

Retrieval quality is not enough in high-stakes settings. A useful RAG system needs source identity, snippet provenance, retryable queries, evaluator traces, and a clear distinction between retrieved evidence and model synthesis. That is the difference between a demo that looks fluent and a workflow an expert can audit.

Field note

Ownership-chain reasoning is not entity extraction

Extracting company names is only the first layer. The hard part is resolving entities across registries, tracking control paths, preserving time and jurisdiction context, and separating direct evidence from inferred links. Graph structure matters because the review question is relational.

Updated Jun 2026

Field note

Evaluation plans should be written before model runs

The fastest way to fool yourself is to inspect results first and decide the metric afterward. For research systems and production ML alike, useful evaluation starts with the claim, the failure mode, the comparison baseline, and the rule for changing direction.
